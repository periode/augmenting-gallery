models of interaction

a good interaction is an exchange between two or more agents (possibly conscious agents), in which the result is greater than the sum of its parts.

(i.e. what has been accomplished could not have been done by only one of the interactors)

In The Art of Interactive Design, Chris Crawford defines an interaction as a conversation, between multiple agents who, one after the other, "listen, think, and speak", which in computer terms, would consist of "input, process, output".

The richness of an interaction comes from the care in the listening experience ("is everything that is being said, being heard?" -think of non-verbal cues), to the depth in processing ("what is it that is really being meant, and how can I extract that meaning to serve as the basis for my response?") to the quality and clarity of the output ("is what I'm saying clearly reflecting the results of my thought process? does it leave room for further exchange?")

Whatever the response/output is, it's always important to leave room for further action. Even when an error happens, the handling of the error should be done "gracefully" (e.g. "This operation did not succeed. Would you like to try again? Or try a different operation?)

some of the requirements for interaction include:

  • being able to communicate exactly what you intend,
  • having an equal interest in both parties,
  • having a common factor, common references,
  • being able to receive information,

links -> recommendations or suggestions

how do these relate to janet murray's components of interaction — interactor, information, agency, transparence ?

Most interactive systems include: the developer, the system and the user.

In terms of the interactor, it's a significant switch from user: user means passive, while interactor means active. Just because one is on the final, receiving end of an event (even if it's an interactive event), doesn't mean that they do not re-appropriate the event for themselves, and engage with it in non-expected ways. There is a thin line between making a system error-proof (making sure the interactor doesn't break the system by inputting the wrong data), and making the system fool-proof (making sure the interactor doesn't do anything too complicated because the assumption is that the interactor can't know as well as the designer).

The concept of agency is tied to the second meaning: an interactive system should enable its interactor, allowing them to do as much as they would like to do, as much as they can imagine, and perhaps even more. A good example of such a thing is the LEGO brick. A system which provides agency is a system which makes you responsible for what you want to be responsible for. On the opposite, a system like an automatic slide door doesn't provide much agency: it's either fully open or fully closed, and doesn't exactly tell you what is need for either of these states to change.

The need for transparency in interactive systems comes from this similar idea. Thinking that a system is intuitive relies on a particular number of assumptions: you already know shortcuts, design heuristics, patterns and processes. A transparent system, on the other side, is a system which exposes itself to the interactor, and only assumes that she would be able to make sense of it, given a good interface. Robert Henke's CBM8032 is a good example of the things someone can do if they're given access to all the relevant parts of a system.

Additionally, interfaces on desktop between Mac, Linux and Windows assume very different behaviours of their users by showing them different amounts of information

interactive media vs. digital media vs. new media?

what makes a computer different in its interactivity?

The computer can "think". That is, it can act and respond somewhat dynamically and contextually. If the programmers can think through the possible future situations that the computer will be confronted with, it will give the impression that the interactor is having a "smart" conversation with the system.

In short, digital media is different from other media in that it can reconfigure itself as it goes. The problem, as we've seen with Licklider the previous week, is that thinking through possible problems in advance turns out to be more complicated than expected. So the answer at this point is either to establish the digital product you have as either close-ended or open-ended. Close-ended means that you are not providing anything beyond the basics of the product can do: the scope is tight, the interactions safe, and the "world" narrow. On the other side, open-ended means that you leave some room for interpretation to the interactor. It might not be as rigorous—in the scientific sense of the term—but it will provide a different kind of engagement, by involving the interactor deeper into the meaning of what she is presented with. Open-ended systems involve some kind of gestalt (the whole is greater than the sum of its parts).

The image above exposes a kind of open-ended system: Joseph Weizenbaum developed a program called ELIZA, a chatbot which would essentially "bounce back" questions that the interactor would input. For instance, if you would type "I'm unhappy", ELIZA would respond "Why are you unhappy?". While the computer program has no idea of what "unhappiness" is, the interaction still elicits interesting thoughts in the interactor's mind.

the specificity of media

how does a medium affect a particular message?

sender <- --- medium ----> receiver

The guy in the picture right there is Marshall McLuhan, who in his books Understanding Media started the field of media studies.

It makes things more or less difficult to say in particular mediums. While there is no "true", "actual" limit to what can be said in a particular way, it's a lot easier to work in the field of classical Western philosophy when you have a pen and a paper than when you can only broadcast smoke signals.

It also affects what receivers pay attention to. The predominance of a particular sense might be hard to overcome (and therefore should be taken into account), when a message is being received.

A newer medium also develops on top of previous media, in a phenomenon called remediation (writing integrates parts of speech, photography integrate parts of drawing, cinema integrates parts of photography, etc.)

dimensions of digital media

-> spatial

-> procedural

-> encyclopedic

-> participatory

(first developed in hamlet on the holodeck)

The spatial component comes from this possibility of navigation, of going from one place to another. There are locations (websites), pathways (hyperlinks), landmarks (icons) which allow us to both explore and keep track of where we are. How do we make sure that the interactor never feels lost?

The procedural component relies on the fact that we are dealing specifically with digital media. Through a computer, it becomes possible to set up rules, procedures, in order to allow for everything else (space, encyclopedia, participation).

The encyclopedic component is the consequence of working with networked media. If one is connected to the Internet, there is no pre-existing reason as to why one shouldn't be able to access any information that is on the Internet. A good example of that is Assassin's Creed's Discovery Mode through which you walk around the game world and learn of the "real world" equivalent of what you're seeing.

interactive digital media

interactive digital media can create conversations with whole worlds, allowing for the co-construction of meaning.

So, as we start developing interactive experiences within the context of museums, it's important to keep in mind those things:

  • How are you enabling an interesting conversation? How responsive and predictive is your system. For instance, if the visitor takes one action (view an object up close), then would be the next action (e.g. viewing other objects? viewing advanced information related to the same object?)
  • What kind of world are you creating? What are its multiple components, and where do you put the emphasis? Is it a social world, an economic world, a world connected to other worlds, etc.?
  • How is meaning co-constructed? How do you let the visitor explore them in more details, at their own pace, in their own way? And to what extent do you ensure that all visitors get the same amount of knowledge, and each visitor gets their own version?

how do we express things through interaction?

live-taped corridor

the other: rest energy

In Bruce Nauman's piece, Live-Taped Corridor, we can identify two obvious main components: the interactor (the person holding the phone in the video) and the system (comprised of the two monitors, the CCTV camera, and the corridor itself).

It's possible to think of the system itself as made up of smaller components, yet the CCTV acts as an input receiver (filming the corridor), the monitors act as the output medium (showing the video), and the corridor itself acts as a scenography, making sure that the interactor enters the system in the "correct" way. Finally, the third component is the artist, or the designer, the person who set up the whole system and whose presence is like a ghost, or an angel, hovering above the piece while remaining transparent and barely visible.

How does a ceremony express importance? How does the way to greet someone in different cultures express particular aspects of those cultures?

How can we tell whether a space is "safe" or not? To what extent does it have to do with the layout of some specific rules? (e.g. codes of conduct?)

This particular piece, by Marina Abramovic and Ulay (1980) works as an artwork since it does ask broader questions from a simple interaction: trust, balance, relationship, etc.

verbal rhetoric: using words to convince

with images

Rhetoric uses logic and (verbal) grammar to get its point across. However, there are other kinds of rhetoric, particularly visual rhetoric and procedural rhetoric. You can make logical connections, emotional connections, and aesthetic connections.

Kenneth Burke's version is: "the use of language as a symbolic means of inducing cooperation in beings that by nature respond to symbols".

Christiane Taubira: "the act we are about to accomplish is as beautiful as a rose, as an eiffel tower besieged at dawn.". This particular quote is powerful because of the linguistic tricks it uses:

  • using the symbol of the rose both as beauty and as the symbol of the socialist party
  • quoting from a french-guyanese poet (Léon Gontran Damas), a black MP whom had to endure adversities in order to sit in Parliament.
  • referring to the Eiffel Tower as France (implying that all of France wants same-sex marriage)
  • referring to the Eiffel Tower being besieged, implying that the conservative party is attacking France by rebuking the law.

Similarly, Barack Obama's speech in 2008 in New Hampshire: "Yes we can.

It was the call of workers who organized; women who reached for the ballot; a President who chose the moon as our new frontier; and a King who took us to the mountaintop and pointed the way to the Promised Land."

This speech made the subtle connection between the US-American people and the Jews of Israel by mentioning the Promised Land.

Finally, rhetoric can also be visual: this cover is taken from an example in Mythologies, a book about the significance of cultural objects. In it, the cover is analyzed as a system of visual signs re-enforcing the colonial vision of the colonized people as a cute child who looks up (implying inferiority) to an idea of France, while pledging allegiance, in a uniform halfway between boy-scouts and military.

expressing with procedures

how do we express things with rules?

An example is the Universal Declarations of Human Rights. There are a set of values, a particular worldview which wants to be expressed by the UN, and such a worldview is going to be realized through rules, through what one can or cannot do.

Other legal systems assume different things about the world, and use rules to solidify these worldviews (e.g. the U.S. Supreme Court granting the same rights to companies as to individuals, interpretations of Sharia law prescribing a bigger share of inheritance to first-born sons over mothers.

a word on games

could everything be a game?

Arguably, everything could be seen as a game. Johan Huizinga's argument in his book Homo Ludens is that, since play isn't restricted to humans, then play actually predates civilization. Is it then possible to think of all civilization as an elaborate game?

If everything has goals and rules, we can think of the U.S. Constitution as the goal of "the pursuit of happiness" and the rules as the Bill of Rights.

In terms of games mimicking real systems, it is interesting to point out the case of The Landlord's Game, a board game originally designed to educate the masses about the evils of real-estate capitalism (i.e. winner-takes-all, others are ruined). The game was then the inspiration of Monopoly, a crude apology of land-grabbing practices.

procedural rhetoric

the art and activity of communicating an idea exclusively through the laying out of rules to be enacted.

the intricacies of being a border patrol agent

it is as if you were doing work

Procedural rhetoric, then, is the ability to develop a system of rules which provides a statement about a particular world, about a particular group of people, about a particular practice, without ever being explicit about that statement.

It is a different way to apprehend a problem: not by listening/reading an argument, not watching a poster or a movie, but instead acting through the consequences of a situation.

the difference

what is the difference between user interface (UI) and user experience (UX) ?

User Experience is comprised of touch points, of particular moments in which the public comes into contact with the product. For instance, the moment that someone books a ticket online for a museum, the moment someone enters the museum and goes through security check, the moment someone asks for directions at the information kiosk, etc. And how all of these come together in a coherent moment.

On the other side, User Interface is the collection of specific graphical elements which help create a metaphor for the interactor to understand how to manipulate and communicate with such a system.

In other words, UI is what we see, and UX is how we feel when we use something (the overarching ceremonial), and how we feel when we're done using it.

the problem

we need to make our product/artefact/app usable, with the lowest cognitive friction, and the highest reliable output.

User interfaces are indeed a metaphor, either building on previous knowledge by the interactors (e.g the Trash on a desktop isn't really a trash, but it does the same thing as a trash does), or by developing new habits (the hamburger icon ).

The point of a good UI is to have itself be forgotten. It should be as invisible as possible, while at the same time being "obvious". This pinnacle of obviousness was reached in Apple products, particularly with the iPod and the iPhone, which didn't come with a user manual!

elements of user interface

no interface is "natural" (example)

-> graphical elements

-> input methods

-> system-related messages

UI elements can be separated in three categories:

  • graphical elements (buttons, icons, layout, typefaces, etc.)
  • input methods (tap, swipe, hold, left-click, right-click, shake, etc.)
  • system messages (confirmation, errors, dialog boxes, alert boxes, etc.)

In their design guidelines for iOS, Apple separates them in three parts:

  • navigation bars
  • action sheets
  • buttons

elements of interaction design

-> visibility (do I see what I'm doing?)

-> feedback (does the system tell me they've acknowledged I've done something?)

-> constraints (does the system prevent me from making basic mistakes?)

-> mapping (is each graphical input element clearly connected to its function(s) ?)

-> consistency (does the input always do the same thing?)

Interaction Design is halfway between UI and UX, it is the art and science of providing the user with an environment in which her actions are meaningful—they are acknowledged and acted upon in a way that is coherent with what she expected.

All of these elements are explained thoroughly in Jenny Preece's Interaction Design reading for this week.

user-centered design

user-centered design is the process of starting from a user, and then modifying the technical system to enable whatever it is that they might think of doing (and more!)

the first question is therefore: who is going to be using my application?

how does one re-design Albert for different users?

-> how would it look like for students?

-> how would it look like for faculty?

-> how would it look like for staff?

-> who else could use it?

Finally, the field of Human-Computer Interaction (HCI) has multiple approaches. One of the predominant of these is User-Centered Design, which starts from the development of a fictional "user" and designs the system around the wants and needs of that user.

One of the most important practices of user-centered design is usability testing, the constant process of making sure that whatever stage of the development process you are at, a given user can still navigate, interact, and understand what is going on!

While this is a better approach that a purely one-way perspective (design the system, and the provide the interface), one of the pitfalls might be that the fictional user whom serves as a base for this design might not actually exist! There is a huge difference between an ideal user and the diversity of individual in practice, and that is something that all designers should remain aware of.

inputs

Input.GetMouseButtonDown(0) is equal to true if the left mouse button has been clicked.

Input.GetTouch(0) gets you a tap.

if(Input.GetMouseButtonDown(0)){
  Debug.Log("you've just clicked!");
}

if(Input.GetTouch(0)){
  Debug.Log("touched at position: "+ Input.GetTouch(0).position);
}

Touch input is actually a little trickier, because there could be multiple touches, pinching, dragging, etc.

This is why we need first to check (with an if-statement) whether there has been a touch, and then what kind of touch it is (do we want the first finger to touch the screen, the second finger, etc.)

materials and images

Images can be Textures or Sprites. You can decide what they are in the Inspector panel.

Sprites are standalone images which can be used directly in the world, and inside UI elements

Textures are images that wrap over an object.

In order to apply an image to an object, you need to create a Material (Assets > Create > Material).

Inside this material, you set your Texture as the Main Maps > Albedo and then set the Material directly on the Renderer component of the GameObject.

In particular, textures are usually a different aspect ration (a power of 2: 512512, 10241024), so while you can technically use a Sprite as a Texture, it might look a little strange (but maybe that's what you want!)

In terms of Materials, an object can have multiple Materials stacked on one another (imagine a "skin tone" material and then a "sun burnt" material on top of it), and Materials can have multiple textures, to simulate depth and shadows.


unity interface

the canvas

All UI elements have to go inside a Canvas object (you can have multiple Canvas objects).

You also need an EventSystem object.

Finally, Canvases can exist in Screen Space or in World Space.

->Screen space means in relation to the two dimensions of your screen.

-> World space means in relation to the three dimensions of the Unity world.

We can think of Canvas objects as special objects which will hold our UI elements (such as buttons, scrollbars, toggles, input fields), and automatically give them the special properties that UI elements usually have:

  • they're directly interactable
  • they're immediately rescaled based on the device screen size and the device orientation

Every UI object (Text, Button, etc.) needs to be the child of a Canvas object.

clickable interface

In order to make an interface clickable, we need to tell it which function to execute. To do so, we need to tell it:

  1. which object holds the script
  2. which script holds the function
  3. which function it should trigger

raycasting

the technique to translate a point on a screen (2D) to a point in a space (3D); it's essentially a computer's version of 15th century perspective.

Raycasting has a long and beautiful story, starting with John Carmack's work on Wolfenstein 3D (first software application with real-time perspective) and Doom (first software application with real-time 3D—meaning you can look up and down). What a raycast does is that, given a point on a screen, and a description of the 3D world, it can figure out what its path in the 3D world will be: is it going to hit something? at which angle? how should the thing that it hits look like? if it's a ray of light, should the object become brighter? if it's a a ray of light blocked by another object, should it make the object behind darker? For our purposes, it allows us to translate between 2D space (the screen) to 3D space (the world).

creating game objects

We want to be able to create Game Objects while our app is running! To do that, we can use Instantiate():

//--this is the gameobject we want to create
public GameObject objectToCreate; 

//-- now we want it to actually show up in the scene
GameObject go = Instantiate(objectToCreate); 

//-- this makes it twice as big!
go.transform.localScale = new Vector3(2, 2, 2);
